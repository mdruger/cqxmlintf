  1. From ClearCase, copy the new files to an empty directory that is accesible from the CQ/XML server.
    • I always create a subdirectory in //indshare/cqxmlintf/release .
    • From the CQ/XML servers, you will need to use standard UNIX paths instead of the UNC path specified above.
      • For example, the previous path would resolve to /indshare/cqxmlintf/release .
    • Henceforth, this directory will be referred to as '[srcdir]'.
  2. Put the server in standby.
    • This prevents users from connecting to the server while the software is being upgraded.
  3. Login to the server with root privileges.
  4. Run /opt/cqxmlintf/cprls2svr.pl -k -d [srcdir]
    • This script will copy the files and stop the currently running CQ/XML Interface process.
    • The CQ/XML Interface will restart within 5 minutes.
