Service_Request Elements

Service_Request elements allow you to view service_request records in ClearQuest. The service_request element has one required and two optional attributes.

Required Attribute Description Notes
id the unique key of the service_request  
Optional Attribute Description Notes
action the action to perform on the service_request
  • valid values: 'view'
  • if the attribute is omitted, 'view' is assumed, but will generate an error if no value is specified (ie: action="")
    		•
    wait Should the action be performed now or queued up for later?
  • valid values: yes, no
  • if the attribute is omitted, 'no' is assumed, but will generate an error if no value is specified (ie: wait="")
    		•

    Service_Request Element Examples

    View service_request output to log: <service_request id='SR10123400'>
    Same as previous example: <service_request id='SR10123400' action='view' wait='no'>
    View service_request, but wait for output: <service_request id='SR10123400' action='view' wait='yes'>

    Putting It Together

    Using the information from the ClearQuest Root Element chapter, logging-in and viewing a service_request would look like this:

       <ClearQuest login='cq_user' password='password' db='pract' repo='practice' email-fail='cq_user@domain.com'>
         <service_request id='SR10123400' action='view' wait='no'/>
       </ClearQuest>

    Caveats

    • Service_Request records can only be read via the CQ/XML Interface.
      • Creating new Service_Request links or modifying existing Service_Request records can only be done through the ClearQuest web interface.
