ClearQuest/XML Interface User Guide

ClearQuest Root Element

Like all well-formed XML, the CQ/XML format contains one root element. The name of that element is 'ClearQuest'. The ClearQuest element is also responsible for passing login information to the ClearQuest server via its attributes. The ClearQuest element has four required and one optional attribute.

Required Attribute Description Notes
login the ClearQuest user login  
password
- or -
encrypted 		the (encrypted) ClearQuest user password

encryption keys must be syncronized with a CQ admin
db the ClearQuest database to login to valid values: pract or prodd
repo the ClearQuest database repository valid values: practice or prodr
Optional Attribute Description Notes
email-fail email address to send errors to on 'queued actions'  

Although the CQ/XML Interface can prohibit actions against defects based on its permissions model, it is still possible to corrupt a large number of defects rather quickly if you are not careful. Therefore, before deploying into production you should do all development and testing against the practice database (db='pract' repo='practice'). If you corrupt data in the production database we will be unable to back out the changes you have introduced.

Since you can only have one root element, to send multiple commands create multiple defect/query/info elements within one root element or make multiple connections to the server. See also: Common Issues, Mistakes & Caveats.

Root Element Examples

Standard login:
   <ClearQuest login='cq_user' password='password' db='pract' repo='practice'>

Login with encrypted password:
   <ClearQuest login='cq_user' encrypted='&±¿þA' db='pract' repo='practice'>

Login with email failure notification:
   <ClearQuest login='cq_user' password='password' db='pract' repo='practice' email-fail='cq_user@mentor.com'>

Caveats

  • Since the 'password' or 'encrypted' attribute is required, it must contain a value. In other words, blank passwords are not permitted.
  • The 'password' or 'encrypted' value cannot contain the greater-than symbol ('>').
