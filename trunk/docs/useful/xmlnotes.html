XML Notes

Naming Tags

  • must start with a letter (A-Z or a-z) or underscore [A-Za-z_]
  • must contain at least one letter
  • can contain letters, digits, hyphens, underscores and periods [A-Za-z_.-]

Well-Formed Documents

  • have one root element
  • no space in the element names
  • all elements must be correctly nested and terminated
  • are case-sensitive
  • enclose all attributes in quotes

Declaration / Processing Instructions

  • the 1st line
  • optional but recommended by the W3C
  • can be used to match documents to parsers
  • PIs are instructions to the parser or agent reading the document
  • not part of the document
  • links XML to other applications

Root

  • must have a unique name
  • links the document to other documents

Element

  • defines the content of the XML doc
  • can contain other elements
  • can contain text (aka "parseable character data" or PCDATA)
  • can be empty

Attribute

  • adds data about the element
  • similar to fields in database tables
  • cannot be repeated

PCDATA

  • parseable character data

CDATA

  • <release><![CDATA[Release 1 & 2]]></release>
  • technique for bypassing the XML parser
  • handles XML reserved characters

Entity

  • predefined shortcut
  • similar to variables
  • XML has predefined entities
