ClearQuest/XML Interface User Guide

Note Elements

Note elements allow you to view note records in ClearQuest. The note element has one required and two optional attributes.

Required Attribute Description Notes
id the unique key of the note  
Optional Attribute Description Notes
action the action to perform on the note
  • valid values: 'view'
  • if the attribute is omitted, 'view' is assumed, but will generate an error if no value is specified (ie: action="")
    		•
    wait Should the action be performed now or queued up for later?
  • valid values: yes, no
  • if the attribute is omitted, 'no' is assumed, but will generate an error if no value is specified (ie: wait="")
    		•

    Note Element Examples

    View note output to log: <note id='pract00301000-0000-000'>
    Same as previous example: <note id='pract00301000-0000-000' action='view' wait='no'>
    View note, but wait for output: <note id='pract00301000-0000-000' action='view' wait='yes'>

    Putting It Together

    Using the information from the ClearQuest Root Element chapter, logging-in and viewing a note would look like this:

       <ClearQuest login='cq_user' password='password' db='pract' repo='practice' email-fail='cq_user@mentor.com'>
         <note id='pract00301000-0000-000' action='view' wait='no'/>
       </ClearQuest>

    Caveats

    • New notes must be created from defect records.
      • To do this, use the 'defect' record and update the 'Note_Entry', 'note_customer', 'note_release' and/or 'note_resolution' fields.
